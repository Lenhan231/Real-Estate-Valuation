# House Price Prediction Models

Machine learning models for predicting Vietnamese house prices from geospatial features (OpenStreetMap) and property characteristics.

## Structure

```text
models/
├── train.ipynb     ← Full workflow: train + predict (Random Forest, Gradient Boosting)
├── predict.py      ← Load best model and make predictions via CLI
├── data/           ← Datasets, all generated by running train.ipynb
│   ├── raw_data.csv            ← Raw fetch from Supabase Raw_Features (all 45 columns)
│   ├── model_ready_data.csv    ← Cleaned training data (17 features + target)
│   └── predictions_latest.csv  ← Predictions for all listings
├── saved_models/   ← Trained model artifacts (best model only)
│   ├── gradient_boosting.joblib
│   └── gradient_boosting_meta.pkl
└── README.md
```

## Usage

### Training (notebook)

Open and run **`models/train.ipynb`** from top to bottom (kernel: `Python (HousePricePrediction .venv)`):

- Part 1 (Training): fetches data from Supabase (`Raw_Features` table), saves `data/raw_data.csv` and `data/model_ready_data.csv`, explores the training data, trains Random Forest + Gradient Boosting, evaluates, visualizes, saves the best model to `saved_models/`.
- Part 2 (Inference): loads the saved model, predicts prices for all listings, computes errors, visualizes, saves results to `data/predictions_latest.csv`.

### Prediction only (CLI)

```bash
python models/predict.py   # load saved model + save predictions
```

## Features Used

**Geospatial (from Overpass API):**

- `nearest_school_km`, `school_count_3km`
- `nearest_hospital_km`, `hospital_count_5km`
- `nearest_marketplace_km`, `marketplace_count_3km`
- `nearest_supermarket_km`, `supermarket_count_3km`
- `nearest_mall_km`, `mall_count_3km`
- `nearest_bus_stop_km`, `bus_stop_count_1km`
- `nearest_metro_km`, `metro_count_5km`

**Property (numeric):**

- `area_m2`, `width_m`, `length_m` - Property dimensions
- `num_floors`, `num_bedrooms` - Layout
- `road_width_m` - Width of the fronting road
- `distance_to_center_km` - Distance to city center
- `locality_population_density` - Population density

**Property (binary, True/False → 1/0):**

- `dining_room_bin`, `kitchen_bin`, `terrace_bin`, `car_parking_bin`, `owner_listing_bin`

**Categorical (text → one-hot encoded):**

- `property_type` (nha_mat_tien / nha_trong_hem)
- `legal_status` (so_hong_so_do / giay_to_hop_le / unknown)
- `direction` (9 compass directions)

**Dropped columns:** `locality`, `region`, `locality_square` (location identifiers), `listing_type` (single value), `link`/`title`/`street`/addresses (free text), `lat`/`lon`.

**Missing-value rules:**

- `nearest_*` → filled with column **max** (missing = no amenity nearby)
- `length_m`/`width_m` → derived from `area_m2 / the other side` when one is missing; median when both are
- `num_floors`, `num_bedrooms`, `road_width_m` → median
- Rows missing `locality_population_density` → dropped

**Target:**

- `price_billion_vnd` - House price in billion VND (computed as `price_vnd / 1e9` from the `Raw_Features` table)

## Model Performance

Evaluated on a 20% held-out test set with:

- **R² Score** - Variance explained (0-1, higher is better)
- **RMSE** - Root Mean Squared Error in billion VND
- **MAE** - Mean Absolute Error in billion VND

The best of the two models (by R²) is saved to `saved_models/` along with a `_meta.pkl` file containing the feature list and metrics.

## Output Files

**After training:**

- `models/saved_models/<best_model>.joblib` - Model binary (e.g. `gradient_boosting.joblib`)
- `models/saved_models/<best_model>_meta.pkl` - Features & metrics

**After prediction:**

- `models/data/predictions_latest.csv` - All input features plus `predicted_price_billion_vnd`, `price_billion_vnd` (actual) and `error_billion_vnd`

## Common Issues

**ModuleNotFoundError (pipeline.supabase_handler)** — run from the project root:

```bash
cd /path/to/HousePricePrediction
python models/predict.py
```

**No trained models found** — run the training notebook first: `models/train.ipynb`

**Supabase connection error** — check `.env` has valid credentials:

```text
SUPABASE_URL=...
SUPABASE_SERVICE_KEY=...
```

## Customization

Edit `train.ipynb` to add features (`FEATURE_COLS`), change hyperparameters (`RandomForestRegressor(...)`, `GradientBoostingRegressor(...)`) or the train/test split (`test_size`).
