# 📊 Data Pipeline Documentation

**Understanding how data flows through the system: raw data → processing → features → model training**

---

## 🔄 Pipeline Overview

```
Web Scraping (alonhadat.com)
    ↓
Raw CSV Files (12,814 properties)
    /data/raw/
    ├── alonhadat_listings.csv      (URLs, basic info)
    └── alonhadat_details.csv       (Full property details)
    
    ↓ [pipeline/ingestion/]
    
Load & Cache in Supabase
    ↓ [pipeline/transformation/]
    
Clean Data (filter outliers, handle missing)
    10,432 properties remain after:
    - Price range: 2-50B VND
    - Area range: 15-500 m²
    - Remove duplicates (260 rows)
    - Remove NULL addresses (1 row)
    
    ↓
/data/processed/
├── alonhadat_cleaned.csv
├── alonhadat_features.csv
└── alonhadat_features_cleaned.csv

    ↓ [pipeline/transformation/feature_pipeline.py]
    
166 Engineered Features
    
    ↓ [Supabase Cache]
    Raw_Features table (1,208 cached locality records)
    
    ↓
/notebooks/02_model_training/
Train LightGBM, XGBoost, CatBoost
Select best model → save to /models/production/
```

## 🔧 Data Processing Stages

### Stage 1: Ingestion (`pipeline/ingestion/`)

**Input:** Raw scraped CSV files  
**Process:** Load from Supabase or local files  
**Output:** DataFrame in memory

```python
from pipeline.supabase_handler import fetch_csv_from_supabase
df = fetch_csv_from_supabase()  # Fetches Raw_Features table
```

**What it handles:**
- Parse CSV files
- Handle missing values
- Basic type conversion

---

### Stage 2: Cleaning (`pipeline/transformation/`)

**Input:** Raw DataFrame (12,814 rows)  
**Process:** Remove outliers, duplicates, invalid data  
**Output:** Clean DataFrame (10,432 rows)

**Cleaning rules:**
```
Price:              2B - 50B VND (removes luxury outliers)
Area:              15 - 500 m²
Price per m²:      30M - 800M VND/m²
Duplicates:        Remove (260 rows)
NULL addresses:    Remove (1 row)
```

**Result:** ~18% data loss, but **much** better model performance

**Code location:** `pipeline/transformation/cleaning.py` & `feature_pipeline.py`

---

### Stage 3: Feature Engineering (`pipeline/transformation/feature_pipeline.py`)

**Input:** Clean DataFrame (10,432 rows × 11 columns)  
**Process:** Generate 155 engineered features  
**Output:** DataFrame (10,432 rows × 166 columns)

#### Base Features (11)
```
Property dimensions:
- area_m2              Area (m²)
- num_floors           Number of stories
- num_bedrooms         Bedroom count
- width_m              Width (m)
- length_m             Length (m)
- road_width_m         Road frontage (m)

Location:
- lat, lon             Coordinates
- distance_to_center   Distance to city center (km)

Category:
- property_type        "nha_mat_tien" or "nha_trong_hem"
- direction            N/S/E/W orientation
- legal_status         Documentation status (Sổ đỏ, etc.)
```

#### Engineered Features (155)

**1. Geometric Features (15)**
```python
perimeter = 2 * (width_m + length_m)
shape_ratio = width_m / length_m
area_x_floors = area_m2 * num_floors
area_per_floor = area_m2 / num_floors
# ... 11 more
```

**2. Temporal Features (3)**
```python
post_day_year, post_day_month, post_day_day
# From posting date
```

**3. Location Features (20)**
```python
location_score = distance_weighted_amenities()
# Distance to center, district, proximity to metro
```

**4. Amenity Features (40)**
```python
school_count              # Within 1 km
hospital_count            # Within 1 km
supermarket_count         # Within 500m
# ... 37 more POI counts and distances
```

**5. Text/Description Features (20)**
```python
is_mat_tien = 1 if "mặt tiền" in description else 0
is_hem_xe_hoi = 1 if "hẻm xe hơi" in description else 0
# Extract keywords from text
```

**6. Interaction Features (30)**
```python
location_x_area = location_score * area_m2
amenity_x_price_tier = amenity_count * is_premium
# Capture relationships between features
```

**7. Log Transforms (10)**
```python
log_area = log(area_m2 + 1)
log_distance = log(distance_to_center + 1)
# Improve model fitting for skewed distributions
```

**8. Categorical (One-hot encoding) (17)**
```python
direction_N, direction_S, direction_E, direction_W
property_type_mat_tien, property_type_hem
# Convert categories to binary features
```

### Stage 4: Caching & Storage

**Supabase `Raw_Features` Table:**
```
Columns:              Purpose
locale_name           Locality name (e.g., "Quận 1")
lat, lon              Coordinates
school_count          Cached POI counts
hospital_count        (to avoid re-querying)
supermarket_count
... (40+ columns)

Records: 1,208 unique localities
```

**Why cache?**
- Avoid re-querying Overpass API (slow)
- Speed up feature engineering
- Store computed location features

**Code:** `pipeline/supabase_handler.py`, `cache_handler.py`

## ⚠️ Common Issues & Solutions

### Issue 1: Feature Count Mismatch
**Error:** `LightGBM failed: expected 166 features, got 150`

**Cause:** Feature engineering pipeline changed or incomplete  
**Fix:**
```python
# Re-run feature engineering
python pipeline/transformation/feature_pipeline.py

# Then retrain model
python notebooks/02_model_training/02_model_training.ipynb
```

### Issue 2: Supabase Connection Error
**Error:** `Could not connect to Supabase`

**Cause:** Missing or wrong credentials  
**Fix:**
```bash
# Check .env file
cat .env
# Should have:
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_SERVICE_KEY=xxx
```

### Issue 3: Data Quality Drop
**Issue:** Model performance suddenly worse

**Cause:** New data has different distribution  
**Debug:**
```python
import pandas as pd
df = pd.read_csv('data/processed/alonhadat_features_cleaned.csv')

# Check distributions
print(df['area_m2'].describe())
print(df['price'].describe())

# Look for outliers
print(df[(df['price'] < 0) | (df['price'] > 100e9)])
```

## 🔄 Updating the Pipeline

**When to update feature engineering:**
1. New data shows different patterns
2. Model performance drops
3. Want to add new features (POI types, transit, etc.)

**Steps:**
1. Modify `/pipeline/transformation/feature_pipeline.py`
2. Re-run: `python pipeline/transformation/feature_pipeline.py`
3. Check output: `/data/processed/alonhadat_features_cleaned.csv`
4. Retrain models: `notebooks/02_model_training/02_model_training.ipynb`
